## 执行摘要

**问题**: PostgreSQL 数据库初始化失败，导致所有服务无法启动

**根本原因**: docker-compose.yml 中 pg_hba.conf 文件在数据库初始化前就被挂载到数据目录，导致数据目录不为空，阻止 PostgreSQL 执行 initdb 命令

**解决方案**: 移除 docker-compose.yml 中 pg_hba.conf 的提前挂载配置

**结果**: ✅ 所有服务成功启动并运行正常

## 实施的修复

### 修复步骤

#### 步骤 1: 停止所有容器

```bash
cd /data/halo
docker-compose down
```

#### 步骤 2: 清空数据库数据目录

```bash
rm -rf db/data/*
```

#### 步骤 3: 修改 docker-compose.yml

**修改文件**: `docker-compose.yml`

**修改内容**: 第 76 行

```yaml
# 修改前
- ./config/database/pg_hba.conf:/var/lib/postgresql/data/pg_hba.conf

# 修改后
# 注意：pg_hba.conf 不再提前挂载，将在初始化后通过 entrypoint 脚本处理
```

#### 步骤 4: 重新启动服务

```bash
make start
```
