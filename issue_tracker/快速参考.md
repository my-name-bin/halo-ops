## 🚨 紧急情况

### PostgreSQL 初始化失败

**症状**:
```
initdb: error: directory "/var/lib/postgresql/data" exists but is not empty
```

**快速解决方案**:

```bash
# 1. 停止所有容器
cd /data/halo
docker-compose down

# 2. 清空数据库数据目录
rm -rf db/data/*

# 3. 重新启动
make start
```

**预计恢复时间**: 1-2 分钟

## 📋 日常检查清单

### 启动前检查

```bash
# 1. 检查数据库目录
ls -la db/data/

# 2. 如果有 pg_hba.conf，手动删除
rm -f db/data/pg_hba.conf

# 3. 检查服务状态
docker-compose ps -a
```

### 启动后验证

```bash
# 1. 检查所有容器健康状态
docker-compose ps

# 2. 查看日志
docker-compose logs --tail=20

# 3. 测试访问
curl -I https://gb.aace.cc
```



## ⚠️ 重要注意事项

### ❌ 不要做的事

1. **不要在容器运行时删除 db/data/ 目录中的文件**
   - 可能导致数据损坏
   - 先停止容器

2. **不要跳过配置验证步骤**
   - 可能导致部署失败
   - 使用 `make init-env` 检查环境

3. **不要忽略健康检查警告**
   - 问题会逐渐恶化
   - 及时排查

### ✅ 应该做的事

1. **定期备份数据**
   - 使用 `make backup`
   - 至少每周一次

2. **查看日志**
   - 及时发现问题
   - 使用 `docker-compose logs -f`

3. **记录问题**
   - 创建 issue_tracker 条目
   - 便于团队协作
