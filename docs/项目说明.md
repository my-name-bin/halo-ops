# Halo 博客系统

> 基于 Docker Compose 部署的生产级博客系统

## 项目概述

Halo 博客系统是一个使用 Docker Compose 部署的生产级博客解决方案，集成了 PostgreSQL 数据库、Nginx 反向代理和 Let's Encrypt SSL 证书自动化管理。

**核心特点**：

- ✅ 自动化部署（Docker Compose）
- ✅ 通配符 SSL 证书（*.aace.cc）
- ✅ DNS-01 自动验证（DNSPod 集成）
- ✅ 证书自动续期（100% 自动化）
- ✅ 零停机部署（Nginx 热加载）
- ✅ 数据持久化（本地卷挂载）
- ✅ 生产级优化（资源限制、日志管理、性能调优、安全加固）
- ✅ 配置文件全映射（PostgreSQL、Nginx 配置文件映射到宿主机）
- ✅ 环境变量管理（敏感信息与配置分离）

## 快速开始

### 环境要求

- Docker >= 20.10
- Docker Compose >= 2.0
- 系统内存 >= 2GB
- 可用磁盘空间 >= 10GB

### 1. 环境检查

```bash
cd /data/halo
./scripts/init/01-check-env.sh
```

### 2. 配置环境变量

编辑 `.env` 文件：

```bash
# 数据库配置
DB_PASSWORD=your_secure_password
DB_USER=halo
DB_NAME=halo

# Halo 应用配置
HALO_EXTERNAL_URL=https://gb.aace.cc/
HALO_JVM_XMX=384m
HALO_JVM_XMS=384m

# DNSPod API 凭据
DNSPOD_API_ID=your_api_id
DNSPOD_API_TOKEN=your_api_token

# SSL 证书配置
DOMAIN=*.aace.cc
EMAIL=your_email@example.com
```

### 3. 启动服务

```bash
# 启动所有服务
make start

# 或直接使用 docker-compose
docker-compose up -d
```

### 4. 验证部署

```bash
# 检查服务状态
make status

# 健康检查
make health

# 查看日志
make logs
```

## 运维命令

使用 Makefile 管理所有运维操作。如果是初学者，请先阅读 [Makefile初学者学习指南.md](Makefile初学者学习指南.md)。

### 服务管理

```bash
make start              # 启动所有服务
make stop               # 停止所有服务
make restart            # 重启所有服务
make status             # 查看服务状态
```

### 日志管理

```bash
make logs               # 查看所有日志
make logs-halo          # 查看 Halo 日志
make logs-nginx         # 查看 Nginx 日志
make logs-db             # 查看数据库日志
```

### 备份与恢复

```bash
make backup             # 全量备份
make backup-db          # 备份数据库
make backup-app         # 备份应用数据
make backup-ssl          # 备份 SSL 证书
```

### 维护

```bash
make health             # 健康检查
make clean-logs         # 清理日志
make clean-docker       # 清理 Docker 资源
```

### SSL 证书

```bash
make cert-status        # 查看证书状态
make cert-renew         # 续期证书
```

### 测试

```bash
make test               # 运行所有测试
make test-config         # 测试配置
make test-database       # 测试数据库
make test-app            # 测试应用
make test-ssl            # 测试 SSL
```

## 配置管理

### 修改 Nginx 配置

```bash
# 编辑主配置
vim /data/halo/config/nginx/nginx.conf

# 编辑站点配置
vim /data/halo/config/nginx/sites-available/halo.conf

# 测试配置
docker-compose exec nginx nginx -t

# 重载配置
docker-compose exec nginx nginx -s reload
```

### 修改 PostgreSQL 配置

```bash
# 编辑主配置
vim /data/halo/config/database/postgresql.conf

# 编辑访问控制
vim /data/halo/config/database/pg_hba.conf

# 重启数据库使配置生效
make restart-db
```

### 修改 Halo 配置

Halo 的配置主要通过启动参数和环境变量管理，编辑 `.env` 文件：

```bash
vim /data/halo/.env
make restart
```

## SSL 证书管理

> **详细文档**: 请参阅 [SSL证书自动续期维护手册.md](docs/SSL证书自动续期维护手册.md)

### 查看证书状态

```bash
make cert-status
```

### 手动续期证书

```bash
make cert-renew
```

### 测试续期流程

```bash
certbot renew --dry-run
```

### 证书自动续期机制

- **定时任务**: Crontab 每天 02:00 自动执行
- **验证方式**: DNS-01 (DNSPod)
- **相关脚本**: `scripts/certbot/renew-cert.sh`
- **日志文件**: `logs/halo/halo-cert-renew.log`

## 备份与恢复

### 自动备份

建议配置 Cron 任务自动备份：

```bash
# 每天凌晨 2 点备份
0 2 * * * cd /data/halo && make backup >> /data/halo/logs/backup.log 2>&1
```

### 恢复数据库

```bash
# 从备份恢复
./scripts/restore/restore-database.sh /path/to/backup.sql
```

### 恢复应用数据

```bash
# 从备份恢复
./scripts/restore/restore-app.sh /path/to/backup.tar.gz
```
