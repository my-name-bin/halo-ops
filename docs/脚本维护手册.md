## 概述

本文档详细描述了 Halo 博客系统项目中所有脚本的功能、用法和维护指南。所有脚本均采用 Bash Shell 编写，遵循项目脚本规范。

### 脚本存放位置

```
/data/halo/
├── scripts/
│   ├── init/                  # 初始化脚本
│   │   └── 01-check-env.sh    # 环境检查脚本
│   ├── backup/                # 备份脚本
│   │   ├── backup-database.sh # 数据库备份
│   │   ├── backup-app.sh      # 应用数据备份
│   │   └── backup-ssl.sh      # SSL证书备份
│   ├── certbot/               # SSL证书脚本
│   │   ├── renew-cert.sh      # 证书续期主脚本
│   │   ├── deploy-hook.sh     # 证书部署钩子
│   │   └── hooks/             # DNS验证钩子
│   │       ├── dns-auth.sh    # DNS认证
│   │       └── dns-cleanup.sh # DNS清理
│   ├── maintenance/           # 维护脚本
│   │   └── check-health.sh    # 健康检查
│   └── utils/                 # 工具函数库
│       ├── color.sh           # 彩色输出函数
│       ├── logging.sh         # 日志函数
│       └── common.sh          # 公共函数
```

#### 依赖关系

- **环境变量**：无

```bash

print_red "发生错误"

# 使用进度条

    sleep 0.1

```

#### 常见问题

**Q: 输出的颜色在某些终端不显示？**
A: 部分老旧终端不支持 ANSI 颜色代码，建议升级终端模拟器。

**Q: 如何禁用颜色输出？**
A: 设置环境变量 `NO_COLOR=1` 或使用管道重定向输出。

### 2. logging.sh - 日志函数库

#### 基本信息

| 属性 | 值 |
|------|-----|
| **文件名** | logging.sh |
| **路径** | scripts/utils/logging.sh |
| **创建日期** | 2026-05-31 |
| **最后修改** | 2026-05-31 |
| **负责人** | guobin |
| **版本** | v1.0.0 |
| **可执行** | 否（仅函数库） |

#### 功能描述

提供统一的日志记录功能，支持日志级别、文件记录、日志轮转等。

#### 核心逻辑

- 定义日志级别：DEBUG < INFO < SUCCESS < WARNING < ERROR
- 日志同时输出到终端和文件
- 支持日志文件轮转，避免占用过多磁盘空间

#### 函数列表

**日志函数：**
```bash
log_debug "调试信息"       # 调试级别
log_info "提示信息"        # 信息级别
log_success "成功信息"     # 成功级别
log_warning "警告信息"     # 警告级别
log_error "错误信息"       # 错误级别
```

**日志管理函数：**
```bash
init_log_file "脚本名" "子目录"  # 初始化日志文件
rotate_log "日志文件" [最大大小] [保留数量]  # 日志轮转
```

#### 依赖关系

- **外部库**：无
- **内部模块**：
  - color.sh（用于彩色输出）
- **环境变量**：
  - `LOG_DIR` - 日志目录（默认：/data/halo/logs）
  - `LOG_LEVEL` - 日志级别（默认：INFO）
  - `LOG_FILE` - 日志文件路径

#### 使用示例

```bash
#!/bin/bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
source "${SCRIPT_DIR}/utils/logging.sh"

export LOG_FILE="/data/halo/logs/myscript.log"

log_info "开始执行任务"
log_success "任务完成"
log_error "发生错误"
```

#### 常见问题

**Q: 如何设置不同的日志级别？**
A: 设置环境变量 `LOG_LEVEL`，可选值：DEBUG、INFO、SUCCESS、WARNING、ERROR

**Q: 日志文件占用空间过大怎么办？**
A: 使用 `rotate_log` 函数进行日志轮转，默认保留7天的日志。

#### 常见问题

**Q: 提示 "Docker 未安装"？**
A: 执行以下命令安装 Docker：
```bash
curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun

```

**Q: 提示 "磁盘空间不足"？**
A: 清理磁盘空间，确保至少有 5GB 可用空间：
```bash
df -h /data/halo
du -sh /data/halo/*
```

**Q: 提示端口被占用？**
A: 检查并停止占用端口的服务：
```bash
netstat -tuln | grep 80
netstat -tuln | grep 443
```

**Q: 提示环境变量缺失？**
A: 复制环境变量模板并配置：
```bash
cp .env.example .env
vim .env
```

#### 功能描述

备份 PostgreSQL 数据库，支持压缩备份、自动清理旧备份。

#### 核心逻辑

1. 创建备份目录（如果不存在）
2. 使用 pg_dump 导出数据库
3. 可选压缩（使用 gzip）
4. 清理超过保留期限的旧备份
5. 记录备份日志

#### 输入参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-h, --help` | 显示帮助信息 | - |

| `-v, --verbose` | 详细输出模式 | - |

#### 输出

- **标准输出**：备份文件路径
- **日志文件**：logs/backup/backup.log
- **备份文件**：db/backups/halo_db_YYYYMMDD_HHMMSS.sql[.gz]

#### 依赖关系

- **外部命令**：
  - docker
  - docker-compose
  - pg_dump
  - gzip（可选）
- **内部模块**：
  - common.sh
- **环境变量**：
  - DB_USER
  - DB_NAME
  - MAX_BACKUP_COUNT（保留备份数量，默认30）

#### 使用方法

**基本用法（压缩备份）：**
```bash
./scripts/backup/backup-database.sh
```

**不压缩备份：**
```bash
./scripts/backup/backup-database.sh --no-compress
```

**使用 Makefile：**
```bash
make backup-db
```

#### 常见问题

**Q: 备份失败，提示权限不足？**
A: 确保 docker-compose.yml 中的数据库用户配置正确，容器内有 pg_dump 命令。

**Q: 如何设置备份保留数量？**
A: 设置环境变量 MAX_BACKUP_COUNT：
```bash
export MAX_BACKUP_COUNT=10
./scripts/backup/backup-database.sh
```

**Q: 备份文件占用空间过大？**
A: 使用压缩备份（默认开启），或定期清理旧备份。

### 1. renew-cert.sh - 证书续期主脚本

#### 基本信息

| 属性 | 值 |

| **负责人** | guobin |
| **版本** | v1.0.0 |

#### 功能描述

自动化续期 Let's Encrypt SSL 证书，使用 DNS-01 验证方式。

#### 核心逻辑

1. **自动安装 certbot**：如果未安装，自动检测包管理器并安装
2. **检查必要文件**：验证 DNS 钩子和部署钩子脚本
3. **执行 certbot renew**：调用 Let's Encrypt API 续期证书
4. **部署证书**：调用部署钩子，将证书复制到 Nginx 配置目录
5. **重载 Nginx**：平滑重载 Nginx 配置，不中断服务

#### 输入参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 无 | 脚本自动执行，无需参数 | - |

#### 输出

- **标准输出**：操作进度和结果
- **日志文件**：logs/halo/halo-cert-renew.log

#### 依赖关系

- **外部命令**：
  - certbot（自动安装）
  - apt-get/yum/dnf/pacman（自动安装时使用）
- **内部模块**：
  - dns-auth.sh（DNS认证钩子）
  - dns-cleanup.sh（DNS清理钩子）
  - deploy-hook.sh（证书部署钩子）
- **配置文件**：
  - .env（DNSPod API 凭据）
  - ssl/live/aace.cc（证书目录）

#### 使用方法

**手动执行：**
```bash
./scripts/certbot/renew-cert.sh
```

**使用 Makefile：**
```bash
make cert-renew
```

**测试模式（不真正续期）：**
```bash
certbot renew --dry-run
```

#### 常见问题

**Q: 脚本提示 "certbot 未安装"？**
A: 脚本会自动检测包管理器并安装 certbot，无需手动安装。

**Q: DNS 验证失败？**
A: 检查以下内容：
1. DNSPod API 凭据是否正确配置在 .env 文件中
2. 域名 DNS 解析是否已生效
3. 等待 DNS 传播（可能需要 5-30 分钟）

**Q: 证书续期成功但 Nginx 未重载？**
A: 检查 deploy-hook.sh 是否正确执行，手动重载：
```bash
docker-compose exec nginx nginx -s reload
```

## 常见问题

### 环境问题

**Q: 如何解决 "command not found" 错误？**
A: 确保已安装必要的命令：
```bash
# Ubuntu/Debian
apt-get install docker.io docker-compose curl openssl netcat-openbsd

# CentOS/RHEL
yum install docker docker-compose curl openssl nc
```

**Q: 如何解决权限不足问题？**
A: 将用户添加到 docker 组：
```bash
sudo usermod -aG docker $USER
newgrp docker
```

### 备份问题

**Q: 备份失败如何排查？**
A: 检查以下内容：
1. 磁盘空间是否充足
2. 备份目录权限是否正确
3. 数据库连接是否正常
4. 查看详细日志

**Q: 如何设置自动备份？**
A: 配置 Cron 任务：
```bash
# 编辑 crontab
crontab -e

# 添加备份任务（每天凌晨2点）
0 2 * * * cd /data/halo && make backup >> /data/halo/logs/backup.log 2>&1
```

### SSL证书问题

**Q: 证书申请失败如何处理？**
A: 按以下步骤排查：
1. 确认域名已正确解析
2. 检查 DNSPod API 凭据
3. 等待 DNS 传播（5-30分钟）
4. 查看证书日志
```bash
cat logs/halo/halo-cert-renew.log
```

**Q: 如何手动强制续期证书？**
A: 使用 certbot 强制续期：
```bash
certbot renew --force-renewal
```

### 健康检查问题

**Q: 健康检查脚本返回失败但服务实际正常？**
A: 检查网络连通性和端口占用：
```bash
netstat -tuln | grep 80
netstat -tuln | grep 443
```

**Q: 如何设置定时健康检查？**
A: 配置 Cron 任务：
```bash
# 每小时检查一次
0 * * * * cd /data/halo && make health >> /data/halo/logs/health.log 2>&1
```
