## 一、项目概览

Halo 博客系统是一个基于 Docker Compose 部署的生产级博客解决方案，集成了 PostgreSQL 数据库、Nginx 反向代理和 Let's Encrypt SSL 证书自动化管理。

### 1.1 项目信息

| 项目 | 内容 |
|------|------|
| 项目名称 | Halo 博客系统 |
| 项目路径 | `/data/halo` |
| Halo 版本 | 2.24.0 |
| PostgreSQL 版本 | 15.4 |
| Nginx 版本 | 1.27-alpine |
| Docker Compose 版本 | 1.29.2 |

## 四、关键文件说明

### 4.1 核心配置文件

#### docker-compose.yml

**位置**: `/data/halo/docker-compose.yml`

**说明**: Docker Compose 核心配置文件，定义了三个服务：
- `halo` - Halo 主应用服务
- `halodb` - PostgreSQL 数据库服务
- `nginx` - Nginx 反向代理服务

**关键配置**:
- 服务依赖关系（depends_on）
- 卷挂载路径
- 环境变量
- 健康检查
- 资源限制
- 日志配置

#### .env

**位置**: `/data/halo/.env`

**说明**: 环境变量配置文件，包含敏感信息：
- 数据库密码
- Halo 配置参数
- DNSPod API 凭据
- SSL 证书配置

**注意**: 此文件不应提交到版本控制系统

### 4.2 数据库配置

#### PostgreSQL 主配置

**位置**: `/data/halo/config/database/postgres/postgresql.conf`

**说明**: PostgreSQL 主配置文件，控制数据库行为：
- 连接配置
- 内存配置
- 写入配置
- 日志配置

#### 访问控制配置

**位置**: `/data/halo/config/database/pg_hba.conf`

**说明**: PostgreSQL 客户端认证配置文件，定义谁可以连接到哪个数据库。

### 4.3 Nginx 配置

#### 主配置文件

**位置**: `/data/halo/config/nginx/nginx.conf`

**说明**: Nginx 主配置文件，定义全局设置：
- 工作进程数
- 事件处理
- 日志格式
- Gzip 压缩
- 安全响应头

#### 站点配置

**位置**: `/data/halo/config/nginx/sites-available/halo.conf`

**说明**: Halo 博客的站点配置，定义：
- SSL 证书路径
- 反向代理规则
- 访问限制
- 缓存配置

## 五、数据流说明

### 5.1 请求处理流程

```
用户请求 (HTTPS)
    ↓
Nginx (端口 443)
    ↓ 验证 SSL 证书
    ↓ 反向代理到 Halo
Halo 应用 (端口 8090)
    ↓ 处理业务逻辑
    ↓ 读写数据
PostgreSQL (端口 5432)
    ↓ 数据持久化
存储 (db/data/)
```

### 5.2 备份数据流

```
备份触发 (定时/手动)
    ↓
backup-database.sh → db/backups/    (SQL 备份)
backup-app.sh → data/app/backups/  (应用备份)
backup-ssl.sh → ssl/backups/       (证书备份)
```

## 六、运维命令速查

### 6.1 服务管理

```bash
make start              # 启动所有服务
make stop               # 停止所有服务
make restart           # 重启所有服务
make status            # 查看服务状态
```


## 八、扩展说明

### 8.1 添加新服务

如需添加新服务，编辑 `docker-compose.yml`：

```yaml
services:
  new_service:
    image: new_service:version
    # ... 配置
```

### 8.2 自定义配置

#### 自定义 Halo 配置

在 `.env` 中添加或修改 Halo 环境变量。

#### 自定义 Nginx 配置

编辑 `config/nginx/` 下的配置文件。

#### 自定义 PostgreSQL 配置

编辑 `config/database/postgres/postgresql.conf`。

### 8.3 备份策略建议

**每日备份**:
```bash
# 每天凌晨 2 点全量备份
0 2 * * * cd /data/halo && make backup
```

**每周清理**:
```bash
# 每周清理超过 30 天的备份
0 3 * * 0 find /data/halo/db/backups -mtime +30 -delete
```
